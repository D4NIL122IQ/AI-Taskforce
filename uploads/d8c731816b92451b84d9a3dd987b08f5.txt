CHAPITRE 1 : FONDEMENTS DE L'INFORMATIQUE
-----------------------------------------------------------

1.1 Histoire de l'informatique

L'informatique est née au milieu du XXe siècle. En 1936, Alan Turing publie son concept de machine universelle, posant les bases théoriques de l'informatique moderne. En 1945, John von Neumann propose l'architecture qui porte son nom : une mémoire centrale, une unité de traitement, des entrées/sorties. Cette architecture est encore utilisée aujourd'hui dans la quasi-totalité des ordinateurs.

Les premières machines comme l'ENIAC (1945) occupaient des salles entières et consommaient des milliers de watts. Avec l'invention du transistor en 1947, puis des circuits intégrés dans les années 1960, les ordinateurs ont pu se miniaturiser. La loi de Moore, énoncée en 1965, prédit que le nombre de transistors sur une puce double tous les deux ans.

Les années 1980 ont vu l'émergence des ordinateurs personnels (PC) avec des systèmes comme MS-DOS puis Windows. Internet est devenu accessible au grand public dans les années 1990, transformant radicalement la société et l'économie mondiale.

1.2 Architecture des ordinateurs

Un ordinateur moderne est composé de plusieurs éléments fondamentaux :

Processeur (CPU)
Le processeur est le cerveau de l'ordinateur. Il exécute les instructions des programmes. Les processeurs modernes contiennent plusieurs coeurs (cores) permettant l'exécution parallèle de tâches. Les caractéristiques principales sont :
- Fréquence (GHz) : nombre de cycles par seconde
- Nombre de coeurs : 4, 8, 16, 32 coeurs pour les processeurs haut de gamme
- Cache : mémoire ultra-rapide intégrée au processeur (L1, L2, L3)
- Architecture : x86, ARM, RISC-V

Mémoire RAM
La RAM (Random Access Memory) est une mémoire volatile qui stocke les données en cours d'utilisation. Elle est beaucoup plus rapide que le disque dur mais perd son contenu à l'extinction. Les types courants sont DDR4 et DDR5. Une machine moderne dispose généralement de 8 à 64 Go de RAM.

Stockage
- HDD (Hard Disk Drive) : disque mécanique, lent mais bon marché
- SSD (Solid State Drive) : flash memory, rapide et silencieux
- NVMe : SSD connecté directement au bus PCIe, extrêmement rapide

Carte graphique (GPU)
Le GPU (Graphics Processing Unit) était initialement conçu pour l'affichage. Il contient des milliers de petits coeurs spécialisés dans les calculs parallèles. Aujourd'hui il est massivement utilisé pour l'entraînement des modèles d'IA. NVIDIA, AMD et Intel sont les principaux fabricants.

1.3 Systèmes d'exploitation

Un système d'exploitation (OS) est le logiciel fondamental qui gère les ressources matérielles et fournit des services aux applications.

Windows
Développé par Microsoft, Windows est le système le plus utilisé sur les ordinateurs personnels. Windows 11 est la version actuelle. Il offre une interface graphique intuitive et une large compatibilité logicielle.

Linux
Linux est un système open source basé sur le noyau créé par Linus Torvalds en 1991. Il existe de nombreuses distributions : Ubuntu, Debian, Fedora, Arch Linux, CentOS. Linux est dominant sur les serveurs, les superordinateurs et les systèmes embarqués.

macOS
Développé par Apple, macOS équipe les ordinateurs Mac. Basé sur Unix, il offre stabilité et performance, particulièrement apprécié des développeurs et créatifs.

Android et iOS
Sur les appareils mobiles, Android (Google) et iOS (Apple) dominent le marché. Android est basé sur le noyau Linux.

CHAPITRE 2 : PROGRAMMATION ET LANGAGES
-----------------------------------------------------------

2.1 Paradigmes de programmation

Programmation impérative
Le code décrit étape par étape ce que la machine doit faire. C'est le paradigme le plus ancien et le plus direct. Langages : C, Pascal, BASIC.

Programmation orientée objet (POO)
Le code est organisé autour d'objets qui regroupent données (attributs) et comportements (méthodes). Les concepts clés sont :
- Encapsulation : cacher les détails internes
- Héritage : une classe hérite des propriétés d'une autre
- Polymorphisme : un même nom de méthode pour des comportements différents
- Abstraction : simplifier la complexité
Langages POO : Java, Python, C++, C#, Ruby.

Programmation fonctionnelle
Les fonctions sont des citoyens de première classe. On évite les effets de bord et les états mutables. Ce paradigme facilite le raisonnement sur le code et la parallélisation.
Langages : Haskell, Erlang, Clojure, Scala. Python et JavaScript supportent aussi ce paradigme partiellement.

2.2 Langages de programmation populaires

Python
Créé par Guido van Rossum en 1991, Python est aujourd'hui le langage le plus populaire au monde. Ses atouts :
- Syntaxe simple et lisible
- Vaste écosystème de bibliothèques (NumPy, Pandas, TensorFlow, PyTorch)
- Idéal pour la data science, l'IA, le scripting, le web (Django, FastAPI)
- Interprété, dynamiquement typé
- Communauté immense

JavaScript
Langage du web par excellence, JavaScript s'exécute dans le navigateur mais aussi côté serveur avec Node.js. Frameworks populaires : React, Vue.js, Angular (frontend), Express, NestJS (backend). TypeScript est une version typée de JavaScript très appréciée des équipes.

Java
Langage fortement typé, orienté objet, compilé vers du bytecode exécuté par la JVM (Java Virtual Machine). Très utilisé en entreprise, pour les applications Android, et les systèmes distribués. Frameworks : Spring Boot, Hibernate.

Rust
Langage moderne axé sur la sécurité mémoire sans garbage collector. Très apprécié pour les systèmes performants et sûrs. Mozilla, Microsoft et Google l'adoptent de plus en plus.

Go (Golang)
Créé par Google, Go est simple, performant et excellent pour les services réseau et les microservices. Son système de goroutines facilite la concurrence.

2.3 Structures de données et algorithmes

Tableau (Array)
Collection d'éléments de même type stockés en mémoire contiguë. Accès en O(1) par index, insertion/suppression en O(n).

Liste chaînée (Linked List)
Chaque élément (noeud) pointe vers le suivant. Insertion/suppression en O(1) en tête, parcours en O(n).

Pile (Stack)
Structure LIFO (Last In First Out). Opérations push et pop. Utilisée pour la gestion des appels de fonctions, l'évaluation d'expressions.

File (Queue)
Structure FIFO (First In First Out). Opérations enqueue et dequeue. Utilisée dans les files d'attente, les systèmes de messagerie.

Table de hachage (Hash Map)
Structure associant clés et valeurs. Accès, insertion, suppression en O(1) en moyenne. Très utilisée dans les caches, les bases de données, les compilateurs.

Algorithmes de tri
- Bubble Sort : O(n²), pédagogique mais inefficace
- Quick Sort : O(n log n) en moyenne, très utilisé en pratique
- Merge Sort : O(n log n) garanti, stable
- Heap Sort : O(n log n), en place

Algorithmes de recherche
- Recherche linéaire : O(n)
- Recherche binaire : O(log n), nécessite un tableau trié
- BFS (Breadth First Search) : parcours en largeur d'un graphe
- DFS (Depth First Search) : parcours en profondeur d'un graphe
- Algorithme de Dijkstra : plus court chemin dans un graphe pondéré

CHAPITRE 3 : RÉSEAUX ET INTERNET
-----------------------------------------------------------

3.1 Modèle OSI et TCP/IP

Le modèle OSI (Open Systems Interconnection) décrit les communications réseau en 7 couches :
1. Physique : bits, câbles, signaux électriques
2. Liaison de données : trames, MAC address, Ethernet
3. Réseau : paquets, IP, routage
4. Transport : segments, TCP/UDP, contrôle de flux
5. Session : établissement et gestion des sessions
6. Présentation : encodage, chiffrement, compression
7. Application : HTTP, FTP, SMTP, DNS

3.2 Protocoles essentiels

HTTP et HTTPS
HTTP (HyperText Transfer Protocol) est le protocole du web. HTTPS ajoute une couche de chiffrement TLS/SSL. Méthodes HTTP :
- GET : récupérer une ressource
- POST : envoyer des données
- PUT : mettre à jour une ressource
- DELETE : supprimer une ressource
- PATCH : mise à jour partielle

DNS
Le DNS (Domain Name System) traduit les noms de domaine (google.com) en adresses IP (142.250.74.46). C'est l'annuaire d'Internet.

TCP vs UDP
TCP : orienté connexion, garantit la livraison et l'ordre des paquets. Utilisé pour le web, les emails.
UDP : sans connexion, plus rapide mais sans garantie. Utilisé pour la vidéo en streaming, les jeux en ligne.

3.3 Architecture web moderne

Frontend : HTML, CSS, JavaScript. Frameworks : React, Vue.js, Angular, Svelte.

Backend : Node.js, Python (FastAPI, Django, Flask), Java (Spring Boot), PHP (Laravel).

API REST : architecture basée sur HTTP, ressources identifiées par URL.
GraphQL : permet au client de demander exactement les données dont il a besoin.

Microservices : architecture décomposant une application en petits services indépendants. Avantages : scalabilité, déploiement indépendant. Inconvénients : complexité opérationnelle.

CHAPITRE 4 : BASES DE DONNÉES
-----------------------------------------------------------

4.1 Bases de données relationnelles (SQL)

Les bases de données relationnelles organisent les données en tables. Le langage SQL permet de les interroger.

Commandes SQL fondamentales :
- CREATE TABLE : créer une table
- SELECT : lire des données
- INSERT : insérer des données
- UPDATE : modifier des données
- DELETE : supprimer des données

Concepts clés :
- Clé primaire (PRIMARY KEY) : identifiant unique d'une ligne
- Clé étrangère (FOREIGN KEY) : référence vers une autre table
- Index : structure accélérant les recherches
- Jointures : INNER JOIN, LEFT JOIN, RIGHT JOIN
- Transactions ACID : Atomicité, Cohérence, Isolation, Durabilité

Systèmes populaires :
- PostgreSQL : open source, très complet, robuste
- MySQL / MariaDB : populaire pour le web
- SQLite : léger, fichier unique, idéal pour les tests
- Oracle : entreprise, très performant
- Microsoft SQL Server : intégration Windows

4.2 Bases de données NoSQL

Document (MongoDB, CouchDB) : stocke des documents JSON. Flexible, pas de schéma fixe.

Clé-valeur (Redis, DynamoDB) : extrêmement rapide. Utilisé pour les caches, les sessions.

Colonnes (Cassandra, HBase) : optimisé pour l'écriture massive. Utilisé pour les logs, les métriques.

Graphe (Neo4j, Amazon Neptune) : stocke des noeuds et relations. Idéal pour les réseaux sociaux, la détection de fraude.

4.3 ORM (Object-Relational Mapping)

Un ORM permet d'interagir avec une base de données via des objets du langage de programmation.

Exemples :
- SQLAlchemy (Python) : très flexible, utilisé avec FastAPI
- Django ORM (Python) : intégré à Django
- Hibernate (Java) : standard Java EE
- Sequelize (Node.js) : populaire pour les apps Express
- Prisma (TypeScript) : moderne, type-safe

CHAPITRE 5 : SÉCURITÉ INFORMATIQUE
-----------------------------------------------------------

5.1 Triade CIA

- Confidentialité : seules les personnes autorisées peuvent accéder aux données
- Intégrité : les données ne sont pas altérées de manière non autorisée
- Disponibilité : les systèmes et données sont accessibles quand nécessaire

5.2 Types d'attaques courantes

Injection SQL : insertion de code SQL malveillant dans un formulaire. Protection : requêtes paramétrées.

XSS (Cross-Site Scripting) : injection de scripts malveillants dans des pages web. Protection : échapper les sorties HTML.

CSRF (Cross-Site Request Forgery) : forcer un utilisateur authentifié à exécuter des actions non désirées. Protection : tokens CSRF.

Phishing : usurpation d'identité par email. Protection : authentification à deux facteurs.

DDoS : saturation d'un serveur avec des milliers de requêtes. Protection : CDN, rate limiting, WAF.

Ransomware : logiciel malveillant qui chiffre les fichiers et demande une rançon. Protection : sauvegardes régulières.

5.3 Cryptographie

Chiffrement symétrique : même clé pour chiffrer et déchiffrer. Algorithmes : AES, ChaCha20.

Chiffrement asymétrique : clé publique (chiffrement) et clé privée (déchiffrement). Algorithmes : RSA, ECC. Utilisé pour HTTPS, signatures numériques.

Fonctions de hachage : transforme une donnée en empreinte de taille fixe. Non réversible. Algorithmes : SHA-256, bcrypt (pour les mots de passe).

JWT (JSON Web Token) : standard pour transmettre des informations de manière sécurisée. Très utilisé pour l'authentification dans les API REST.

CHAPITRE 7 : INTELLIGENCE ARTIFICIELLE AVANCÉE
-----------------------------------------------------------

7.1 Machine Learning en détail

Préparation des données :
- Collecte : APIs, scraping, bases de données
- Nettoyage : valeurs manquantes, doublons, outliers
- Transformation : normalisation, standardisation, encodage
- Feature engineering : création de nouvelles variables
- Split : train/validation/test (ex: 70/15/15)

Métriques de classification :
- Accuracy : taux de bonne classification
- Précision : vrais positifs / (vrais positifs + faux positifs)
- Rappel : vrais positifs / (vrais positifs + faux négatifs)
- F1-score : moyenne harmonique précision/rappel
- AUC-ROC : aire sous la courbe ROC

Métriques de régression :
- MAE (Mean Absolute Error)
- MSE (Mean Squared Error)
- RMSE (Root Mean Squared Error)
- R² (coefficient de détermination)

Problèmes courants :
- Underfitting : modèle trop simple, ne capture pas les patterns
- Overfitting : modèle trop complexe, mémorise les données d'entraînement

7.2 Deep Learning en détail

CNN (Convolutional Neural Networks) :
Spécialisés dans le traitement d'images. Couches : Convolution, Pooling, Flatten + Dense.
Applications : classification d'images, détection d'objets (YOLO), segmentation sémantique.

RNN / LSTM / GRU :
Traitent les séquences en mémorisant l'état précédent. LSTM résout le problème du gradient vanishing.
Utilisés pour : séries temporelles, NLP, génération de texte.

Transformers :
Architecture révolutionnaire introduite en 2017 ("Attention is All You Need"). Basée sur le mécanisme d'attention. Backbone de tous les LLM modernes.
Composants : tokenisation, embeddings, positional encoding, self-attention, multi-head attention, feed-forward network, layer normalization.

7.3 NLP (Natural Language Processing)

Tâches NLP :
- Classification de texte : sentiment analysis, détection de spam
- Named Entity Recognition (NER) : identifier personnes, lieux, dates
- Traduction automatique : Neural Machine Translation
- Résumé automatique : extractif ou abstractif
- Question Answering : répondre à des questions sur un texte

Pipeline NLP :
1. Tokenisation
2. Normalisation : minuscules, suppression accents
3. Stopwords removal
4. Stemming/Lemmatisation
5. Vectorisation : TF-IDF, Word2Vec, BERT embeddings

Bibliothèques NLP :
- NLTK : classique, pédagogique
- spaCy : industriel, rapide
- Hugging Face Transformers : état de l'art, modèles pré-entraînés

7.4 RAG (Retrieval-Augmented Generation) avancé

Architecture complète d'un système RAG :

Ingestion (Indexation) :
1. Chargement des documents (PDF, TXT, HTML, Markdown, DOCX)
2. Découpage en chunks : RecursiveCharacterTextSplitter, chunk_size=500, overlap=50
3. Génération des embeddings : nomic-embed-text, text-embedding-ada-002
4. Stockage dans la base vectorielle : Chroma, FAISS, Pinecone

Query (Interrogation) :
1. Réception de la question utilisateur
2. Transformation en embedding
3. Recherche des K chunks les plus similaires (cosine similarity)
4. Construction du prompt avec le contexte récupéré
5. Génération de la réponse par le LLM

Optimisations avancées :

Chunking strategies :
- Fixed-size chunking : découpage par nombre de tokens
- Recursive chunking : respecte la structure du texte
- Semantic chunking : regroupe les phrases sémantiquement proches

Retrieval strategies :
- Dense retrieval : recherche par similarité vectorielle
- Sparse retrieval : BM25, TF-IDF, recherche par mots-clés
- Hybrid retrieval : combine dense et sparse
- MMR (Maximal Marginal Relevance) : diversité des résultats
- Reranking : second passage pour reclasser les résultats

Évaluation d'un RAG :
- Faithfulness : la réponse est-elle fidèle au contexte ?
- Answer relevancy : la réponse répond-elle à la question ?
- Context precision : les chunks récupérés sont-ils pertinents ?
- Context recall : tous les chunks pertinents ont-ils été récupérés ?
Frameworks d'évaluation : RAGAS, TruLens

7.5 Agents IA

Un agent IA est un système qui perçoit son environnement, prend des décisions et exécute des actions.

Composants d'un agent :
- LLM : cerveau de l'agent, raisonnement et planification
- Tools : fonctions que l'agent peut appeler (recherche web, calcul, API)
- Memory : mémoire à court terme (contexte) et long terme (base vectorielle)
- Planner : décomposition des tâches complexes

Frameworks multi-agents :
- LangChain : framework de référence pour les applications LLM
- LangGraph : orchestration de graphes d'agents avec état
- AutoGen (Microsoft) : agents conversationnels multi-agents
- CrewAI : agents avec rôles et objectifs définis

Patterns d'agents :
- ReAct (Reasoning + Acting) : raisonnement et action alternés
- Chain of Thought : chaîne de réflexion avant d'agir
- Plan and Execute : planifier puis exécuter étape par étape
- Self-reflection : l'agent évalue et corrige ses propres réponses

7.6 Vector Stores en détail

Chroma : base vectorielle open source, facile à utiliser localement. Idéal pour les projets petite/moyenne taille.

FAISS (Facebook AI Similarity Search) : très performante, supporte des milliards de vecteurs. Plusieurs index : Flat, IVF, HNSW.

Pinecone : service cloud managé. Scalable, performant, avec filtrage avancé. Idéal pour la production.

Weaviate : open source avec support GraphQL. Supporte la recherche hybride.

Qdrant : open source écrit en Rust. Très performant, support des filtres avancés.

Métriques de similarité :
- Cosine similarity : angle entre deux vecteurs
- Dot product : produit scalaire
- Euclidean distance : distance L2 entre deux points

CHAPITRE 8 : GÉNIE LOGICIEL
-----------------------------------------------------------

8.1 Méthodes de développement

Méthode en cascade (Waterfall) : approche séquentielle analyse → conception → développement → test → déploiement.

Scrum (Agile) :
- Sprint : itération de 1 à 4 semaines
- Product Backlog : liste des fonctionnalités à développer
- Daily Standup : réunion quotidienne de 15 minutes
- Sprint Review : démonstration au client
- Rétrospective : amélioration continue

Kanban :
- Tableau visuel avec colonnes (To Do, In Progress, Done)
- Limiter le travail en cours (WIP limit)
- Flux continu sans sprints fixes

8.2 Design Patterns

Patterns créationnels :
- Singleton : une seule instance d'une classe
- Factory : création d'objets sans spécifier la classe concrète
- Builder : construction étape par étape d'objets complexes

Patterns structurels :
- Adapter : interface compatible entre classes incompatibles
- Decorator : ajouter des comportements dynamiquement
- Proxy : contrôler l'accès à un objet

Patterns comportementaux :
- Observer : notification automatique des dépendants
- Strategy : algorithmes interchangeables
- Command : encapsuler une requête comme objet

8.3 Tests logiciels

Types de tests :
- Tests unitaires : tester une fonction ou classe isolément
- Tests d'intégration : tester l'interaction entre composants
- Tests end-to-end (E2E) : tester le système complet
- Tests de performance : charge, stress, endurance
- Tests de sécurité : pentest, analyse de vulnérabilités

TDD (Test-Driven Development) : écrire le test avant le code. Cycle : Red → Green → Refactor.

Frameworks de test :
- Python : pytest, unittest
- JavaScript : Jest, Mocha, Cypress (E2E)
- Java : JUnit, Mockito

9.1 Pipeline de données ETL

- Extract : extraire les données des sources (BDD, APIs, fichiers)
- Transform : nettoyer, transformer, enrichir
- Load : charger dans la destination (data warehouse, data lake)

Outils ETL :

9.2 Technologies Big Data

Apache Hadoop : framework pour le traitement distribué. Composants : HDFS (stockage), MapReduce (traitement), YARN (gestion ressources).

Apache Spark : moteur de traitement distribué en mémoire, 100x plus rapide que Hadoop MapReduce. Supporte Python (PySpark). Idéal pour le ML distribué (MLlib).

Apache Kafka : plateforme de streaming distribué. Permet de publier et consommer des flux de messages en temps réel. Utilisé pour les logs, les événements, l'analytics temps réel.

9.3 Bibliothèques Data Science Python

- TensorFlow / Keras : deep learning (Google)
- PyTorch : deep learning (Meta), préféré en recherche

Data Warehouse :
- Snowflake : cloud-native, scalable
- BigQuery (Google) : serverless, très performant
- Redshift (Amazon) : intégré à l'écosystème AWS
- ClickHouse : open source, très rapide pour l'analytics

CHAPITRE 10 : TENDANCES ET FUTUR
-----------------------------------------------------------

10.1 IA générative

L'IA générative est capable de créer du contenu original : texte, images, audio, vidéo, code.

Modèles de diffusion : Stable Diffusion, DALL-E, Midjourney génèrent des images à partir de descriptions textuelles.

Modèles multimodaux : GPT-4V, Gemini, Claude peuvent traiter images et texte simultanément.

Text-to-speech : Whisper (OpenAI) pour la transcription. ElevenLabs pour la synthèse vocale réaliste.

10.2 Informatique quantique

L'informatique quantique exploite les propriétés de la mécanique quantique.

Concepts :
- Qubit : peut être 0 et 1 simultanément (superposition)
- Intrication : corrélation entre qubits
- Interférence : amplifier les bonnes solutions

Applications potentielles :
- Cryptographie : briser les chiffrements actuels (RSA)
- Optimisation : problèmes combinatoires complexes
- Simulation moléculaire : découverte de médicaments

Acteurs : IBM Quantum, Google Quantum AI, IonQ, D-Wave.

10.3 Edge Computing et IoT

Edge Computing : traitement des données au plus près de leur source, réduisant la latence. Crucial pour l'IoT, les véhicules autonomes.

IoT (Internet of Things) : des milliards d'objets connectés génèrent des données en temps réel. Protocoles : MQTT, CoAP, Zigbee.

10.4 Blockchain et Web3

Blockchain : registre distribué, immuable et transparent. Applications : cryptomonnaies (Bitcoin, Ethereum), smart contracts, traçabilité.

Smart contracts : code exécuté automatiquement sur la blockchain. Langage Solidity pour Ethereum.

Web3 : vision d'un internet décentralisé. NFT, DeFi (finance décentralisée), DAO (organisations autonomes décentralisées).
